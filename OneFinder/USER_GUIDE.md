# OneFind v2.0 - 用户指南

## 🚀 快速开始

### 基本使用

1. **启动程序**
   - 运行 `OneFind.exe`
   - 程序会自动连接到 OneNote

2. **搜索内容**
   - 在搜索框输入关键词（如 "async"）
   - 按 `Enter` 或点击"搜索"按钮
   - 等待搜索完成

3. **查看结果**
   - 每个匹配项显示为独立的列表项
   - 包含页面路径和匹配片段
   - 关键词用红色高亮显示

4. **打开匹配**
   - 双击任意结果项
   - 或选中后按 `Enter` 键
   - OneNote 会自动跳转到对应位置

## 📖 界面说明

### 结果列表格式

```
笔记本 › 节 › 页面 [匹配序号/总数]
    ……前文 [关键词] 后文……
```

**示例：**
```
JavaScript › ES6 › 异步编程 [2/5]
    ……错误处理使用 try-catch 包裹 [async/await]……
```

解读：
- `JavaScript`：笔记本名称
- `ES6`：节名称
- `异步编程`：页面名称
- `[2/5]`：这是该页面的第 2 个匹配，共 5 个
- 片段：显示匹配关键词的上下文
- `[async/await]`：高亮的匹配关键词

### 匹配序号说明

| 显示        | 含义                      |
|------------|---------------------------|
| 无标记      | 该页面只有 1 个匹配         |
| `[1/2]`    | 第 1 个匹配，该页面共 2 个   |
| `[2/2]`    | 第 2 个匹配，该页面共 2 个   |
| `[3/10]`   | 第 3 个匹配，该页面共 10 个  |

### 状态栏信息

```
找到 5 个页面，共 12 处匹配（双击打开）
     ↑            ↑
  页面数      总匹配数
```

## 💡 使用技巧

### 1. 精确跳转到每个匹配

**场景：** 同一页面有多个匹配，想逐个查看

**操作：**
1. 在结果列表中找到该页面的所有匹配
2. 双击第 1 个 `[1/N]` → 跳到第一处
3. `Alt+Tab` 回到 OneFind
4. 双击第 2 个 `[2/N]` → 跳到第二处
5. 重复以上步骤查看所有匹配

### 2. 快速对比不同页面

**场景：** 想对比不同页面中对同一概念的描述

**操作：**
1. 搜索关键词（如 "Promise"）
2. 浏览所有匹配的片段预览
3. 从预览中判断哪些页面更相关
4. 双击打开感兴趣的页面

### 3. 利用片段快速筛选

**场景：** 搜索结果很多，想快速找到特定内容

**操作：**
1. 观察每个匹配的片段预览
2. 根据上下文判断是否相关
3. 只打开相关的匹配项
4. 节省时间，避免逐个查看

### 4. 键盘快捷操作

| 按键          | 功能              |
|--------------|-------------------|
| `Enter`      | 开始搜索          |
| `↑` `↓`      | 在结果中移动       |
| `Enter`      | 打开选中的匹配     |
| `Alt+Tab`    | 在 OneFind 和 OneNote 间切换 |

## ⚙️ 高级功能

### 页内导航

当双击匹配项时，OneFind 会尝试：

1. **精确定位**
   - 跳转到页面中的具体对象
   - OneNote 会自动滚动到该位置
   - 并高亮显示对应的文本

2. **降级处理**
   - 如果精确定位失败（对象已删除等）
   - 自动降级为打开整个页面
   - 用户手动在页面中查找

### 搜索进度

搜索过程中，状态栏会显示：
```
正在搜索…
扫描笔记本：JavaScript
扫描笔记本：工作笔记
扫描笔记本：读书笔记
```

可以实时了解搜索进度。

### 取消搜索

- 开始新搜索时，自动取消之前的搜索
- 避免资源浪费
- 提高响应速度

## 🐛 故障排除

### Q: 双击没有反应？

**A:** 可能的原因：
1. OneNote 未运行 → 启动 OneNote
2. 页面已删除 → 在 OneNote 中确认
3. 权限问题 → 以管理员身份运行

### Q: 搜索很慢？

**A:** 正常现象：
- 需要扫描所有笔记本的所有页面
- 取决于笔记本的数量和大小
- 首次搜索可能需要几秒到几十秒

**优化建议：**
- 使用更具体的搜索词
- 关闭不需要的笔记本
- 定期清理回收站

### Q: 找不到某些内容？

**A:** 可能的原因：
1. 内容在加密的节中 → 解锁后重新搜索
2. 内容在回收站 → OneFind 会跳过回收站
3. 拼写错误 → 检查搜索词

### Q: 片段显示不完整？

**A:** 正常设计：
- 片段只显示匹配词前后 30 个字符
- 使用 `……` 表示省略
- 双击打开查看完整内容

### Q: 页内导航不精确？

**A:** OneNote API 限制：
- 某些页面元素可能无法精确定位
- 会自动降级为打开页面
- 这是 OneNote COM API 的已知限制

## 📊 性能优化建议

### 减少搜索时间

1. **关闭不需要的笔记本**
   - 在 OneNote 中关闭
   - 减少扫描范围

2. **清理笔记本**
   - 删除无用页面
   - 清空回收站

3. **使用更精确的关键词**
   - 避免过于宽泛的词（如 "是"、"的"）
   - 使用技术术语或专有名词

### 提高显示速度

- 结果已经优化，无需特殊处理
- 即使有数百个匹配也能流畅显示
